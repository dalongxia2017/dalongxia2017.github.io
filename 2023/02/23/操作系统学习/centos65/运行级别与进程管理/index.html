运行级别

运行级别 说明
0 关机
1 单用户模式
2 无网络支持的多用户模式
3 有网络支持的完全多用户模式，文本界面
4 未使用
5 有网络支持的完全多用户模式，X11界面
6 重启

查看作业

jobs [选项]
常用选项：

  • -p：仅显示进程号
  • -l：同时显示进程号和作业号

作业前后台切换

<ctrl+z>
挂起作业

bg [作业号]
切换作业到后台运行

fg[作业号]
切换作业到前台运行

查看进程

ps [选项]

常用选项：

  • -a：显示所有程序包括其他用户的程序
  • -u：某个用户的进程
  • -x：显示所有程序，不以终端来区分
  • -A：显示所有程序
  • -l：显示详细信息

top指令

top

进入后按q键退出

输出含义解释：

第一行的内容解释：

top - 13:04:57 up 3 days, 20:14, 1 user, load average: 0.45, 0.59, 0.48

内容 含义
13:04:57 表示当前时间
up 3 days, 20:14 系统远行时间，格式为天，时：分
1 user 当前登陆用户数
load average: 0.45, 0.59, 0.48 系统负载，即任务队列的平均长度。 三个数值分别为 1分钟、5分钟、15分钟前到现在的平均值

第二行的内容解释：

Tasks: 509 total, 1 running, 508 sleeping, 0 stopped, 0 zombie

内容 含义
Tasks: 509 total 进程总数
1 running 正在运行的进程数
508 sleeping 睡眠的进程数
0 stopped 停止的进程数
0 zombie 僵尸进程数

第三行的内容解释：

%Cpu(s): 2.0 us, 1.2 sy, 0.0 ni, 96.2 id, 0.0 wa, 0.0 hi, 0.6 si, 0.0 st

内容 含义
2.0 us 用户空间占用CPU百分比
1.2 sy 内核空间占用CPU百分比
0.0 ni 用户进程空间内改变过优先级的进程占用CPU百分比
96.2 id 空闲CPU百分比
0.0 wa 等待输入输出的CPU时间百分比
0.0 hi 硬中断（Hardware IRQ）占用CPU的百分比
0.6 si 软中断（Software Interrupts）占用CPU的百分比
0.0 st 用于有虚拟cpu的情况，用来指示被虚拟机偷掉的cpu时间

第四行的内容解释：

MiB Mem : 64177.9 total, 6461.9 free, 8067.5 used, 49648.5 buff/cache

内容 含义
64177.9 total 物理内存总量
8067.5 used 使用的物理内存总量
6461.9 free 空闲内存总量
49648.5 buffers 用作内核缓存的内存量

第五行的内容解释：

MiB Swap: 16384.0 total, 16377.2 free, 6.8 used. 54429.6 avail Mem

内容 含义
16384.0 total 交换区总量
6.8 used 使用的交换区总量
16377.2 free 空闲交换区总量
54429.6 cached 缓冲的交换区总量

第六行的内容解释：

PID USER PR NI VIRT RES SHR S %CPU %MEM TIME+ COMMAND

列名 含义
PID 进程id
USER 进程所有者的用户名
PR 优先级
NI nice值。负值表示高优先级，正值表示低优先级
VIRT 进程使用的虚拟内存总量，单位kb。VIRT=SWAP+RES
RES 进程使用的、未被换出的物理内存大小，单位kb。RES=CODE+DATA
SHR 共享内存大小，单位kb
S 进程状态。D=不可中断的睡眠状态 R=运行 S=睡眠 T=跟踪/停止 Z=僵尸进程
%CPU 上次更新到现在的CPU时间占用百分比
%MEM 进程使用的物理内存百分比
TIME+ 进程使用的CPU时间总计，单位1/100秒
COMMAND 命令名/命令行

结束进程

kill [信号代码] PID
killall [信号代码] PID
信号代码：

  • -9：发送SIGKILL信号，无信号的kill无效时，可强制终止进程
  • -15：发送SIGTERM信号，一般在-9选项无效时使用的

优先级设置

nice [-优先级数值] 命令
用优先级的数值运行某个命令

renice -优先级数值 选项
常见选项：

  • -p：修改指定pid的进程的优先级
  • -u：修改指定用户的所有进程的优先级
  • -g：修改指定组的所有进程的优先级

at调度

at [选项] [日期时间]

常见选项：

  • -f：指定包含具体指令的任务文件
  • -q：指定新任务的队列名称
  • -l：显示待执行任务的列表
  • -d：删除指定的待执行任务
  • -m：任务执行完成后向用户发送 E-mail

cron调度

编辑位于/etc/crontab的配置文件
